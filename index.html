Welcome to the ERAD 2016 short course on community radar software

This course will take place on October 9th 2016, right before the ERAD 2016 conference. We will introduce the participants to community software packages designed for radar data processing, mainly BALTRAD and wradlib.

These tools can be controlled by using Python, so we'll feature a quick intro to the Python programming language, and endow participants with the basics of how to contribute to community software. For the course, we will provide a platform-independent virtual machine that contains all the required software, data and other course materials.

Changes in course as compared to the original announcement

We had to slightly adjust the course contents in comparison to the original announcement on this page (see also the adjusted curse program below). We will not provide in-depth coverage of the Py-ART package, although we'll feature a quick background. In addition, the course will not be linked to the "WMO Radar Quality Control Quantitative Precipitaton Estimation Inter-comparison Project" (RQQI) meeting anymore. We would like to apologize for the deviation from the original announcement. However, we still have more than enough exciting stuff to address in the course.

Registration

Register at the ERAD Short Course page for Short Course 1. A fee of 50 EURO will be charged including luch and coffee breaks. Beware that the maximum number of participants is limited to 30.

Things you need to prepare

Participants need to bring their own notebook. All you need to do is to install VirtualBox and import the Virtual Machine for the course. Please see http://openradar.github.io for detailed instructions. However, we will send around an e-Mail with instructions to the registered participants, too.
